# Client Guide

The VibeMongo Admin frontend is a Single Page Application (SPA) built with Vue 3, Vite, and Element Plus.

## 📂 Project Structure

```text
client/
├── public/                 # Static assets (favicon, logos)
├── src/
│   ├── assets/             # CSS and global styles
│   ├── components/         # Reusable Vue components
│   │   ├── chat/           # Agent chat sidebar & message UI
│   │   ├── collection/     # Collection view, index management
│   │   ├── connection/     # Connection manager
│   │   ├── dashboard/      # Dashboard and charts
│   │   ├── document/       # Document viewer/editor
│   │   ├── layout/         # App layouts (Header, Sidebar, footer)
│   │   └── utils/          # Shared utility components
│   ├── locales/            # i18n translation files
│   ├── router/             # Vue Router configuration
│   ├── stores/             # Pinia state management
│   ├── utils/              # Helper functions (API, BSON parsing)
│   ├── views/              # Main route views
│   ├── App.vue             # Root component
│   └── main.js             # Application entry point
├── index.html              # HTML template
├── package.json            # Client dependencies
└── vite.config.js          # Vite configuration
```

## 🚀 Key Technologies

- **Vue 3 (Composition API):** Modern, reactive framework.
- **Vite:** Blazing fast build tool. Proxy is configured in `vite.config.js` to route `/api` calls to the backend during development.
- **Element Plus:** Component library used for layout, tables, forms, and dialogs.
- **Pinia:** State management (Connections, active database, active collection, UI state).
- **vue-router:** Client-side routing.
- **vue-i18n:** Localization support.
- **ECharts / Chart.js:** Used dynamically to render charts generated by the AI agent.

## ⚙️ Configuration

The client relies on environment variables injected via Vite.
Ensure `.env` in the root directory contains:
```env
VITE_API_URL=http://localhost:4000
```
*(In production, this is usually empty or relative `/` if served by the same Express backend).*

## 🛠️ Development

To run the client in development mode with Hot Module Replacement (HMR):

```bash
cd client
pnpm install
pnpm run dev
```

The client will be available at `http://localhost:3000`.

## 📦 Building for Production

The client build process bundles the application into static files.

```bash
cd client
pnpm run build
```

This outputs the production-ready assets to `client/dist`. The backend Express server is configured to serve these files automatically when `NODE_ENV=production`.
